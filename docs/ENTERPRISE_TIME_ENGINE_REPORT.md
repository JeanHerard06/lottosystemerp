# Enterprise Time Engine

## Source of truth

- Default application timezone: `America/Port-au-Prince`.
- Each tenant may override it with `tenant_settings.timezone` using an IANA name.
- `TimeService::boot()` applies the same timezone to PHP and the current MySQL/MariaDB connection.
- The database session receives the current UTC offset, avoiding dependency on installed MySQL timezone tables.

## Services

- `app/Services/TimeService.php`
- `app/Services/LotteryTimeService.php`

## Lottery rules

Lottery close and draw times now use the tenant timezone. Automatic closing writes an explicit timestamp generated by `TimeService` instead of relying on server `NOW()`.

## Mobile API

Every JSON response contains:

```json
"meta": {
  "timezone": "America/Port-au-Prince",
  "utc_offset": "-04:00",
  "server_time": "2026-07-11T...-04:00"
}
```

Diagnostic endpoint: `GET /api/mobile/time.php`.

## Tenant configuration

Set `timezone` in `tenant_settings`, for example:

- `America/Port-au-Prince`
- `America/Santo_Domingo`
- `America/New_York`

Never use abbreviations such as EST because they do not safely handle daylight-saving changes.
