# Contributing

## Branches
Use `feature/<module>`, `hotfix/<bug>`, and `release/v1.0-rc1`.

## Commits
Use: `feat:`, `fix:`, `refactor:`, `security:`, `perf:`, `docs:`, `test:`, `chore:`.

## Pull Requests
Every PR must include tenant impact, security review, QA checklist, and documentation updates.

## Definition of Done
A task is done only when code passes syntax checks, tenant isolation is verified, permissions are checked, migrations are included, docs are updated, and the changelog is updated.
